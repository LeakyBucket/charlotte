Charlotte.Views.Compiler

  • Summary
  • Functions

The Views Compiler builds view modules for all the view files found at the specified path.

During development the views will interpret the file on each render. Otherwise the views will be read and converted into functions via the EEx function_from_file macro.

The Views Compiler assumes that Views live in subdirectories at the given root path. The first directory off the root is considered to be the controller name. The filename is assumed to be the action name.

For example:

```


- views
  - users
    - new.eex
    - create.eex
  - special_needs
    - show.eex
    - new.eex
    - create.eex

```

The above structure would result in the following Modules:

```


Charlotte.Views.Users
  - new/1
  - create/1
Charlotte.Views.SpecialNeeds
  - show/1
  - new/1
  - create/1

```

Each view method takes a keyed list of assignment/value pairs.
